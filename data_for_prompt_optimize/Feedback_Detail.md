| รับมือกับ Negative Case | ทำได้ดี เช่นผู้ป่วยร้องขอยา \_ab\_ และ AI ปฏิเสธ และยึดตามการปฏิบัติตาม Guideline ได้อย่างถูกต้อง | ทำได้ดี เช่น ผู้ป่วยขอยาปฏิชีวนะโดยไม่จำเป็น หรือเคสไอเป็นเลือด ระบบสามารถแนะนำแนวทางปฏิบัติที่เหมาะสม พร้อมให้เหตุผลประกอบ และแนะนำส่งต่อโรงพยาบาลเมื่อมี Red Flag | สามารรับมือกับ Negative Case ได้ดี โดยไม่หลงเชื่อตามข้อมูลที่ผู้ใช้พยายามหลอก และสามารถแนะนำแนวทางการรักษาที่เหมาะสมแทน |
| การแนะนำตาม Guideline | AI แนะนำยาที่เหมาะสมกับโรคได้ตรงตาม Guideline และเมื่อผู้ใช้ไม่ต้องการใช้ยาตัวเดิม ก็สามารถเสนอทางเลือกอื่นได้ | แนะนำตาม First-line Guideline ได้ พร้อมแสดงขนาดยาปกติ ขนาดยาสูงสุด (Min-Max) และคำนวณ Dose ได้อย่างถูกต้อง | แนะนำยา First-line ได้ถูกต้อง พร้อมคำนวณขนาดยาเด็กและระยะเวลาการรักษาได้สอดคล้องกับ Guideline |
| เคสผู้ป่วยแพ้ยา | AI สามารถเลือกยาทดแทนได้อย่างเหมาะสม พร้อมกับขนาดยาที่เหมาะสม | AI สามารถแนะนำยาทดแทนได้อย่างเหมาะสม พร้อมมีตัวเลือกหลายรายการ | AI สามารถแนะนำยาทางเลือกได้อย่างเหมาะสม |
| ยาที่ไม่มีใน Guideline | ระบบจะไม่ตอบเกินขอบเขต แต่แนะนำตาม Guideline เป็นหลัก พร้อมอธิบายการใช้ยานอก Guideline บางกรณี |  | หากข้อมูลหรือคำตอบไม่มีอยู่ใน Guideline ระบบจะสามารถแจ้งได้ว่าไม่มีข้อมูลในเอกสารอ้างอิง |
| คำถามโรคนอกขอบเขตโรคติดเชื้อทางเดินหายใจส่วนบน (นอกขอบเขต Guideline) | AI ยังสามารถวินิจฉัยและให้แนวทางการรักษาได้ค่อนข้างถูกต้อง | ระบบสามารถระบุได้ว่าเป็น “คำถามความรู้ทั่วไป” พร้อมให้ข้อมูลและแนบแหล่งอ้างอิงจากภายนอก | สามารถแยกได้ว่าคำถามเป็นโรคนอกขอบเขตของโรคติดเชื้อทางเดินหายใจส่วนบนและไม่สับสนกับโรคในระบบ |
| แหล่งอ้างอิง (Reference) |  | ทุกคำตอบมีการอ้างอิง แยกเป็นแต่ละส่วน และสามารถกดดูแหล่งอ้างอิงได้ เพิ่มความน่าเชื่อถือของคำตอบ | มีการอ้างอิง Guideline ในคำตอบและสามารถกดเปิดไฟล์อ้างอิงเพื่อตรวจสอบข้อมูลเพิ่มเติมได้ |
| ขนาดยา |  |  | AI ตอบคำถามได้ตรงประเด็นที่ถาม พร้อมคำนวณขนาดยา (Dose) ได้ถูกต้องแม้จะมี Drug Calculator แยกต่างหาก |

| **Reference & Guideline** |  |  |  |
| การอ้างอิงจาก Guideline | คำตอบเกี่ยวกับโรคติดเชื้อทางเดินหายใจส่วนบนควรระบุให้ชัดเจนว่าอ้างอิงจาก Guideline ฉบับใด (อาจมีแนวทางการรักษาที่แตกต่างกัน) ระบุให้ชัดเจน | บางกรณีอ้างอิง Guideline ไม่ตรงกับกลุ่มผู้ป่วย เช่น ผู้ป่วยผู้ใหญ่แต่ดึง Guideline ของเด็กมาใช้อ้างอิง | บางคำตอบไม่มีอยู่ใน Guideline ทั้งที่ข้อมูลดังกล่าวอยู่ในเอกสาร |
|  |  |  | บางคำตอบดึงข้อมูลผิดตำแหน่งใน Guideline ทำให้แม้จะเลือกแนวทางการรักษาถูก แต่เหตุผลหรือรายละเอียดที่อธิบายไม่ถูกต้อง |
| เลขหน้า Guideline |  | เลขหน้าและเนื้อหาอ้างอิงบางส่วนยังไม่ตรงกับ Guideline จริง | เลขหน้าอ้างอิง (Reference) ยังคลาดเคลื่อนหลายกรณี และบางครั้งอ้างอิงข้ามโรค ทำให้ตรวจสอบข้อมูลได้ยาก |
|  |  |  | เลขหน้าของ Guideline AAFP ที่แสดงในระบบไม่ตรงกับเลขหน้าจริงในไฟล์ PDF |
| การอ้างจากนอกขอบเขต | คำถามนอกขอบเขค AI ตอบได้ แต่ต้องดึงอ้างอิง URL จาก internet ด้วย (อ้างอิงข้อมูล) | Ref ที่มาจากภายนอกยังไม่ระบุ URL หรือแหล่งที่มาอย่างชัดเจน |  |
|  | ข้อมูลจากภายนอก ต้องแยกอย่างชัดเจนให้ผู้ใช้ทราบว่าไม่ใช่ข้อมูลจากทาง Guideline |  |  |
| การผสมข้อมูล (ข้อมูลนอก \+ Guideline) |  | บางคำตอบผสมข้อมูลจาก Guideline และข้อมูลทั่วไป ทำให้ผู้ใช้เข้าใจว่าเป็นข้อมูลจาก Guideline ทั้งหมด |  |
| **Clinical Accuracy** |  |  |  |
| ความถูกต้องของข้อมูล | ยังมีผิดอยู่ เช่น AI ระบุว่า **Azithromycin** เป็นยาที่ควบคุมพิเศษที่ต้องใช้ใบสั่งแพทย์ ทั้งที่ในทางปฏิบัติเป็นยาอันตรายที่เภสัชกรสามารถจ่ายได้ |  | AI ยังมีความเข้าใจคลาดเคลื่อนในบางเรื่อง เช่น เข้าใจว่ายาแก้อักเสบคือยาปฏิชีวนะ |
| เหตุผลประกอบ |  |  | เหตุผลประกอบการเลือกแนวทางการรักษาบางกรณียังไม่ถูกต้อง เช่น การเลือกขนาดยา **Amoxicillin-clavulanate** ในผู้ป่วย Acute Otitis Media โดยอธิบายเหตุผลไม่ตรงกับ Guideline |
| ขนาดยา (Dose) | แม้แนะนำ First-line ไดถูกต้อง แต่ขนาดยา (Dose) บางกรณียังไม่สอดคล้องกับ Guideline เช่น แนะนำ **Amoxicillin 500 mg วันละ 3 ครั้ง** ในขณะที่ Guideline แนะนำ**วันละ 2 ครั้ง** หรือควรแนะนำเป็นช่วง **วันละ 2-3 ครั้ง**ตามความรุนแรงของอาการและลักษณะผู้ป่วย | AI คำนวณ Dose เด็กเฉพาะค่าสูงสุด (Maximum Dose) ควรแสดงช่วง Min-Max เพื่อให้เภสัชกรเลือกใช้ขนาดยาที่มีในร้านได้สะดวก |  |
|  | การคำนวณขนาดยายังควรคำนึงถึงความรุนแรงของโรคและลักษณะของผู้ป่วยแต่ละรายมากขึ้น เพื่อให้คำแนะนำมีความยืดหยุ่นและเหมาะสมกับการใช้งานจริง |  |  |
| ขนาดยาน้ำ และหน่วย mL |  | ในอนาคตควรเพิ่มการรับค่าความแรงของยาน้ำ (เช่น Amoxicillin) และคำนวณปริมาณที่ต้องรับประทานเป็น mL ให้อัติโนมัติ |  |
| ระยะเวลาการรักษา |  | บางเคส (เช่น Acute Otitis Media) ยังอ้างอิง Guideline ไม่ครบถ้วน ควรเปรียบเทียบ Guideline ไทยและ AAFP เพื่อให้ข้อมูลครอบคลุมมากขึ้น |  |
| ดึงข้อมูลผิด |  | บางคำตอบยังดึงข้อมูลจาก Guideline ผิดหัวข้อ ทำให้คำตอบเรื่องการใช้ Antibiotic และเลขหน้าอ้างอิงไม่ถูกต้อง (เช่น Sinusitis) | แม้ AI จะตอบยาทางเลือกได้ แต่บางคำตอบยังดึงข้อมูลผิดตำแหน่งจาก Guideline ทำให้รายละเอียดของคำตอบไม่ถูกต้อง |
| เคสเด็ก |  | ในผู้ป่วยเด็กบางกรณี AI ยังแนะนำยาที่ไม่สอดคล้องกับ Guideline เช่น การแนะนำยาแก้ไอในเด็กอายุน้อยกว่า 4 ปี |  |
| ครอบคลุม \+ ครบ |  |  | บางข้อมูลการรักษาตามอาการ (Symptomatic Treatment) ที่มีอยู่ใ น Guideline ยังตอบได้ไม่ครบ |
| **Conversation & Context** |  |  |  |
| การต่อบทสนทนา | เมื่อเป็นการต่อบทสนทนาเดิม AI มักตอบข้อมูลทั้งหมดซ้ำอีกครั้ง ผู้ใช้งานต้องการให้ตอบเฉพาะประเด็นที่ถามเพิ่มเติม เพื่อให้บทสนทนากระชับมากขึ้น | การจดจำบริบทของเคสเดิมยังไม่เสถียร บางครั้งระบบไม่แสดงว่าเป็นการสนทนาต่อเนื่องจากเคสเดิม |  |
| Hallucination | บางกรณี เช่น ระบุว่าผู้ป่วยมีอาการแพ้ยา ทั้งที่ผู้ใช้งานยังไม่เคยแจ้งข้อมูลดังกล่าว | บางกรณี AI แนะนำเพียง “กลุ่มยา” โดยไม่ระบุชื่อยาที่ควรใช้ ต้องถามซ้ำจึงจะแสดงชื่อยา |  |
| การจัดประเภทบทสนทนา |  | การจัดประเภทบทสนทนา (Patient Consultation / General Knowledge) ยังไม่สม่ำเสมอ บางครั้งคำถามต่อเนื่องของเคสผู้ป่วยถูกจัด “เป็นความรู้ทั่วไป” |  |
| Red Flag Case |  | AI ยังแนะนำให้พบแพทย์ในบางกรณีที่เภสัชกรสามารถตัดสินใจรักษาได้ตาม Guideline ควรสงวนคำแนะนำส่งต่อไว้เฉพาะกรณีที่มี Red Flag |  |
| การซักประวัติ เมื่อ context ไม่ครบ |  |  | แม้ AI จะถามคำถามเพิ่มเติมเมื่อข้อมูลไม่ครบ แต่ควรอธิบายเหตุผลของแต่ละคำถาม เช่น ถามระยะเวลาที่เป็นโรคเพื่อประเมินความรุนแรง หรือถามลักษณะอาการแพ้ยาเพื่อใช้เลือกยาทดแทนได้อย่างเหมาะสม |
|  |  |  | เมื่อถามประวัติแพ้ยา ควรถามต่อว่าผู้ป่วยแพ้ยาอย่างไร หรือมีอาการแพ้แบบใดเพื่อใช้ประกอบการตัดสินใจเลือกยา |
