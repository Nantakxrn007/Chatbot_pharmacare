# **Pharmacy Bot Evaluation Checklist**

ตารางเช็คลิสต์สำหรับประเมินความถูกต้องและประสิทธิภาพของ Bot Answering ในบริบทการดูแลผู้ป่วยร้านยาโดยเภสัชกร

| หมวดหมู่ (Category) | หัวข้อย่อยและเกณฑ์การตรวจรับ (Detailed Criteria) | ผลการเช็ค | หมายเหตุ / หลักฐาน   |
| :---- | :---- | :---: | :---- |
| **1\. ความถูกต้อง (Accuracy & Guidelines)** | วินิจฉัย เลือกยา First-line ยาทางเลือกกรณีแพ้ยา ขนาดยา และระยะเวลารักษา ถูกต้องตาม Guideline สม่ำเสมอทุกเคส | \[ \] |  |
|  | ข้อมูลประเภทยาตามกฎหมายถูกต้อง (ยาอันตราย vs ยาควบคุมพิเศษ) เพราะกระทบการตัดสินใจจ่ายยาโดยตรง | \[ \] |  |
|  | คำแนะนำไม่ขัด Guideline โดยเฉพาะประเด็นความปลอดภัยในเด็กเล็ก | \[ \] |  |
|  | ดึงข้อมูลถูกโรค ถูกช่วงวัยผู้ป่วย และถูกหัวข้อใน Guideline | \[ \] |  |
|  | ระบุชื่อตัวยาชัดเจนตั้งแต่คำตอบแรก ไม่ตอบแค่กลุ่มยา | \[ \] |  |
|  | เข้าใจคำที่สะกดต่างหรือใช้คำต่างกันเล็กน้อยแต่ความหมายเดียวกัน (Synonyms) | \[ \] |  |
| **2\. การคำนวณขนาดยา (Dose Calculation)** | แสดง Dose เป็นช่วง Min–Max เต็มช่วงตาม Guideline | \[ \] |  |
|  | คำนวณขนาดยาตามน้ำหนักตัวให้อัตโนมัติ และคำนวณออกมาเป็นช่วงขนาดยา | \[ \] |  |
|  | ระยะเวลารักษาจำเพาะกับผู้ป่วยแต่ละราย (เช่น ตามอายุ) | \[ \] |  |
|  | ยืดหยุ่นตามความรุนแรงของอาการ และแนะนำในรูปแบบใช้จริง เช่น "Amoxicillin 500 mg ครั้งละ 1 เม็ด วันละ 2–3 ครั้ง" | \[ \] |  |
|  | ต่อยอด Drug Calculator: เลือกความแรงยาน้ำที่มีในร้าน และคำนวณปริมาณที่ต้องกินเป็น mL | \[ \] |  |
| **3\. การอ้างอิงแหล่งข้อมูล (References)** | ทุกคำตอบต้องมีเรฟ Guideline กำกับ พร้อมชื่อเอกสารและเลขหน้าที่ถูกต้องแม่นยำ | \[ \] |  |
|  | แยกชัดว่าข้อมูลมาจาก Guideline หรือแหล่งภายนอก ถ้ามาจากอินเทอร์เน็ตให้แปะ URL/ลิงก์ที่มา | \[ \] |  |
|  | ค้นเจอข้อมูลที่มีอยู่ใน Guideline ได้ครบ ต้องไม่ตอบว่า "ไม่มี" ทั้งที่มีอยู่จริง | \[ \] |  |
|  | ใช้แหล่งอ้างอิงครบทุก Guideline (ทั้งแนวทางของไทย และ AAFP) | \[ \] |  |
|  | \[แผนในอนาคต\] แสดงคำแนะนำเปรียบเทียบจากหลาย Guideline ให้เภสัชกรใช้ประกอบการตัดสินใจ | \[ \] |  |
| **4\. การซักประวัติและเหตุผล (History Taking & Clinical Reasoning)** | ซักประวัติครบถ้วน พร้อมเหตุผลกำกับทุกคำถามว่าถามไปเพื่ออะไร (เช่น "เป็นกี่วัน — เพื่อประเมินความรุนแรง") และถามประวัติแพ้ยาพร้อมลักษณะอาการแพ้ | \[ \] |  |
|  | อธิบายเหตุผลประกอบทุกการตัดสินใจ ทั้งการเลือกยา การไม่จ่ายยาปฏิชีวนะ และการส่งต่อโรงพยาบาล | \[ \] |  |
|  | เหตุผลทางคลินิกถูกต้องตามหลักวิชาการ ไม่ใช่แค่คำตอบสุดท้ายถูกอย่างเดียว | \[ \] |  |
|  | เข้าใจบริบทว่าผู้ใช้คือ "เภสัชกร" เน้นเสนอแนวทางรักษาสำหรับร้านยา และแนะนำ "ไปพบแพทย์" เฉพาะกรณีมี Red Flag เท่านั้น | \[ \] |  |
| **5\. รูปแบบคำตอบ (Answer Format)** | เรียงคำตอบเป็นมาตรฐาน 5 ขั้น: สรุปอาการ → วินิจฉัยเบื้องต้น (พร้อม range ความน่าจะเป็นของโรค) → การรักษา (แยกยาปฏิชีวนะ / ยาตามอาการ) → คำแนะนำดูแลตัวเอง → Red Flags | \[ \] |  |
|  | คงการแบ่งบูลเล็ทการตัดสินใจของเภสัชกรพร้อมเหตุผลไว้อย่างชัดเจน | \[ \] |  |
| **6\. การจัดการบทสนทนา (Conversation Context)** | จำบริบทบทสนทนาต่อเนื่อง แยกเคสเดิม-เคสใหม่ได้ถูกต้อง รวมถึงโรคใหม่ในผู้ป่วยคนเดิม | \[ \] |  |
|  | ตอบคำถาม Follow-up เฉพาะประเด็นที่ถามเพิ่ม ไม่ตอบข้อมูลเดิมซ้ำทั้งหมด | \[ \] |  |
|  | จำแนกประเภทคำถามแม่นยำ (เคสผู้ป่วย / ความรู้ทั่วไป / นอกขอบเขต) | \[ \] |  |
|  | ไม่ Hallucinate ไม่สรุปหรือเติมข้อมูลที่ผู้ใช้ไม่ได้ให้มา | \[ \] |  |
|  | คงความระมัดระวังต่อเคสหลอก (Trick questions) และการแยกอาการแพ้ยาจริง/ไม่จริงไว้ | \[ \] |  |
